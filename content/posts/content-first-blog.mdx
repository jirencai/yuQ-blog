# Building a Content-First Blog

When a blog starts simple, the architecture should stay simple.

## Why this structure works

- Content stays in dedicated files.
- Pages focus on rendering.
- The lib layer owns loading and transformation.

This keeps future migrations practical without paying complexity early.

You can read the [Next.js docs](https://nextjs.org/docs) to keep **delivery quality** while preserving *implementation simplicity*.

## Minimal content query flow

```ts
const posts = getAllPostsMeta();
const post = getPostBySlug(slug);
```

1. Keep content in `content/posts`.
2. Keep parsing in `lib/posts`.
3. Keep pages focused on rendering.
